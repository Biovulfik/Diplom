# Тест – план для мобильного приложения «Мобильный хоспис».

## 1. Введение

**Цель тестирования:** Проверить корректность работы реализованного функционала мобильного приложения, включая авторизацию пользователей, управление новостями, отображение цитат и просмотр информации о приложении.

**Объект тестирования:** Мобильное приложение «Мобильный хоспис» с функционалом:

- Авторизация пользователей
- Просмотр и управление новостями
- Добавление, редактирование и удаление новостей
- Отображение цитат
- Просмотр информации о приложении

## 2. Область тестирования

**Функциональное тестирование**

- Авторизация пользователей
- Управление новостями (просмотр, добавление, фильтр, редактирование, удаление)
- Отображение цитат
- Просмотр информации о приложении

**Нефункциональное тестирование**

- Удобство использования

## 3. Подход к тестированию

- **Ручное тестирование:** Проверка вех сценариев использования в ручную.
- **Автоматизированное тестирование:** Написание автотестов для критически важных функций.
- **Тестирование в эмуляторах:** Для проверки базовой функциональности.

## 4. Техники тест-дизайна с обоснованием

**1. Эквивалентное разделение:**

- Для проверки валидных/невалидных логинов и паролей

**2. Анализ граничных значений:**

- Проверка полей с ограничениями
- Проверка пустых полей

**3. Позитивные/негативные сценарии:**

- Проверка что система работает правильно при корректных данных
- Проверка что система не ломается и корректно обрабатывает ошибки

**4. State Transition:**

- Проверка перехода между экранами

## 7. Критерии приемки

- Все функциональные сценарии работают корректно.
- Отсутствие критических багов.

## 8. Перечень и описание возможных рисков при автоматизации

**1. Хрупкость тестов:**

- Тесты могут ломаться при изменениях UI

**2. Зависимость от окружения:**

- Зависимость тестов от доступности внешних URL

**3. Проблемы с синхронизацией:**

- Необходимость Idling Resourse

**4. Ограничения эмуляторов:**

- Проверка интентов (переходов по ссылкам) может вести себя иначе на реальных устройствах

## 9. Интервальная оценка с учетом рисков 

Для подготовки тестовых данных, ручного тестирования, автоматизации тестов и подготовки отчетности необходимо 17 дней.
